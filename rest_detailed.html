1. Write a JAX-RS 2.0 'Hello World' service

First steps, detailed

    • Implement a 'Hello World' Representational State Transfer (REST) service in 1+3 tiny classes, using JAX-RS 2.0, implemented by Jersey 2.

The 'Hello World' REST service will be bundled in a jar (yes, no war). Let's create a new Eclipse project completely independent from the first abendrot-test project used in part 1 (getting started) and part 2 (identity) of this tutorial.

  • Right-click in Eclipse Package Explorer
  • New
  • Other ...
  • Open Maven Folder
  • Select Maven Project
  • Click Next >
  • Check 'Create a Simple Project'
  • Click Next >
  • Enter GroupId ArtifactId
  • Click Finish
Ugly, isn't it ?

Start test-driven development. Add junit4 dependency.

  • Right-click on project 'abendrot-rest' in Package Explorer
  • Maven
  • Add Dependency
  • Type junit in 'Enter ... '
  • Wait while m2e search the repositories indexes
  • Scroll down to folder junit, open it
  • Select a version > 4.8.2
  • Click OK

Stil ugly, isn't it ? It's the green arrow, I believe.

Create new test class in directory src/test/java

Paste the test into the freshly created test class: 

public class ResourcesTest {

    @Test
    public void test() {
        assertEquals("Hello, World!", new Resource().getIt());
    }

}
CTRL-SHIFT-o to let eclipse resolve the java imports. After that "new Resources()" is still underlined in red.
  • Hover over "new Resources()"
  • wait for eclipse to popup the tooltip
  • click "Create class 'Resources'"
  • change the source folder from 'abendrot-rest/src/test/java' to 'abendrot-rest/src/main/java'
  • click Finish

Back to test class. Now, ".getIt()" is underlined in red.

  • Hover over ".getIt()"
  • wait for eclipse to popup the tooltip
  • click "Create method 'getIt() in class 'Resources'"
  • change return type to String
  • save (CTRL-S)

Back to test class. Now, "assertEquals(..." is underlined in red.

  • Hover over"
  • wait for eclipse to popup the tooltip
  • choose your preferite static import. Mine is "Add static import 'org.junit.Assert.assertEquals'"

Run the test.

  • Right-click on ResourceTest in Package Exlorer
  • Run As
  • JUnit Test

Test Fails.

Change method getIt() to return "Hello, World!" instead of null. Re-run the test. Test Passes.

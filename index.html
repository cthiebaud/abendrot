1. Setup SAP NetWeaver Cloud

Open SAP NetWeaver Cloud Developer Center, get your free developer license (in 5 minutes), and perform all the steps covered under the "Step by step" section.

2. Install Maven and Git in Eclipse

SAP NetWeaver Cloud is build technology and source code management system agnostic, hence these two topics are not covered in the SAP step by step guide from section 1. Abenrot uses maven as build technology and git as source code management system; Update your eclipse environment to support maven and git:

Maven Integration for WTP (m2e-wtp) : open https://www.eclipse.org/m2e-wtp/. Follow the installation procedure at https://www.eclipse.org/m2e-wtp/download/. Install the latest release.

Command line maven (optional, but strongly encouraged) : open http://maven.apache.org/. Follow the installation procedure at http://maven.apache.org/download.html. Install the latest release.

Eclipse Git (egit): open http://www.eclipse.org/egit/. Follow the installation procedure at http://www.eclipse.org/egit/download/. Install the latest release.

3. Create a Github Repository

abendrot is a tutorial on how to write a SAP NetWeaver Cloud application. You will use a github repository to host and monitor your work. You will also clone abendrot itself to another directory. And you will use a good compare tool to diff the two directories at various stages of the tutorial.

Setup a free github account, if you have not already one

Log to your account at https://github.com/<your account>

Create an empty github repository

To create an empty Github repository, this click icon in the top right toolbar:

Check 'Initialize this repository with a README' and choose the maven .gitignore.

Create and clone immediately your repository

I assume you've read Github docs such as Bootcamp, and that you know how to clone the repo.

You'll get something like this on your box:

  <repository name>/
  ├── .gitignore
  └── README.md

Clone abendrot from https://github.com/cthiebaud/abendrot.git; checkout the initial state, i.e. the first commit

E.g., using the git command line, changing the default directory name from abendrot to abendrot-clone :

$ git clone https://github.com/cthiebaud/abendrot.git abendrot-clone 
$ cd abendrot-clone
$ git checkout 4d427b99f3b60947e3535e045cdf4b35f979ed7f

3. Create Eclipse Project

There are ~12 different flavors of 'how-to-create-a-maven-aware-eclipse-project'. This one has my preference :

Open eclipse, switch to java perspective

Start the wizard to create a new dynamic Web project

  • File
  • New
  • Other...
  • scroll down and open folder 'Web'
  • Dynamic Web Project
  • click Next >

Fill the project name (e.g. 'abendrot-test'), set project location to your Github clone, accept all other defaults, click Next >

Change the sources directory from eclipse default src to maven default src/main/java

then

Change the output folder from eclipse default build/classes to maven default target/classes, click Next >

Change the webapp folder from eclipse default webContent to maven default src/main/webapp, click Finish

Be polite and say 'no, thanks', when you are invited to change perspective.

Convert to maven Project. Set a groupID (mine is net.aequologica), artifactID (suggested 'abendrot-test'), and packaging type 'war'.

Right-click on the project name in the package explorer to open the context menu, select

  • Configure
  • Convert to Maven Project

Start the wizard to create a HTML page

  • File
  • New
  • Other...
  • scroll down and open folder 'Web'
  • HTML File
  • click Next >

Select the directory 'abendrot-test/src/main/webapp, name it index.html (fancy, isn't it?), click Next >

Select HTML5 format, click Finish

4. Run on local Tomcat

We will run the 'Hello World' index.html page on three different servers: 1. a plain Tomcat, 2. a local test server for SAP NetWeaver Cloud, and 3. the actual SAP NetWeaver Cloud on the internet.

    'Official' SAP NetWeaver Cloud tutorials are usually targetting only NetWeaver Cloud servers 2. (local) and 3. (on the cloud). We need server 1. (plain Tomcat) for two reasons:
  • We aim to have zero coupling with NetWeaver proprietary APIs. Tomcat plain server will tell us if we have.
  • The development cycle with Tomcat is still faster than with Local NetWeaver.

Get Tomcat 7: open http://tomcat.apache.org/. download .tar.gz or .zip file from http://tomcat.apache.org/download-70.cgi. Unzip it to the directory of your choice, we'll reference it in this page as $TOMCAT_HOME

Start 'Run As' wizard on the index.html file we've just created

  • Right click on index.html
  • Run As
  • Run on Server

Select 'Manually define a new Tomcat server', choose Tomcat7, click Next >

Point the wizard to the $TOMCAT_HOME you've just downloaded

Click Finish, Eclipse starts Tomcat and open the page in its internal browser. The page is empty.

Add <h1>Hello , World!</h1> to index.html, save, back to the browser, refresh, the content is displayed, no restart needed.

5. Run on NetWeaver Cloud

Steps to run on both local and remote NetWeaver Cloud are fully documented in the 'Deploying from the Eclipse IDE' page of the SAP NetWeaver Cloud Developer Center. Just use 'index.html' instead of 'HelloWorldServlet'

6. Submit your work to Github with EGit

Connect Eclipse to your git local repository with EGit. Review the Egit Getting Started User Guide.

Right-click on the project name in the package explorer to open the context menu, select

  • Team
  • Share Project ...

Guess what ? Select Git Source Code Management System, then select your local repo.

Give EGit some time to digest the git repo, and the Egit decorations will eventually appear in the package explorer.

Add eclipse configuration files to .gitignore file.

They contain elements of your local configuration that you must not share; and some files in the .settings directory are continously updated by eclipse with timestamps, which will lead to an infinite source of conflicts when stored in git. 

target/
.settings/
.classpath
.project

Commit to local git, then push to remoter git on Github.

Checkout state #2 of abendrot, i.e. commit 42da300...

E.g., using the git command line

HEAD is now at 42da300... Merge branch 'master' of https://github.com/cthiebaud/abendrot.git

7. Continuous integration with travis-ci

Open http://about.travis-ci.org/docs/user/getting-started/. Sign for a free membership with your Github identity. Follow the process for a java project using maven :

Create a .travis.yml file in your project top directory with the following content:

language: java
jdk:
  - oraclejdk7

Commit and push to Github.

Open and login to https://travis-ci.org/profile# and hook your project to travis.ci

Trigger your first travis-ci build with a push to Github. You can have a peek to how I did it
